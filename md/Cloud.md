# クラウドコンピューティング概要と学習パス

## 1. クラウドとは何か

**クラウド（Cloud Computing）**とは、  
サーバー・ストレージ・データベース・ネットワークなどの IT リソースを  
**インターネット経由で、必要な分だけ利用できる仕組み**です。

### 従来（オンプレミス）との違い

| 項目 | オンプレミス | クラウド |
|---|---|---|
| 初期費用 | 高い（サーバ購入） | 低い |
| 拡張性 | 低い | 非常に高い |
| 運用 | 自前 | マネージド |
| 利用開始 | 数週間〜 | 数分〜 |

---

## 2. クラウドの主な特徴

### オンデマンド利用
- 必要なときに即時利用
- サーバー起動・停止が自由

### スケーラビリティ
- アクセス増加時に自動スケール
- トラフィックに強い設計が可能

### 従量課金
- 使った分だけ支払う
- 小規模スタートが容易

### マネージドサービス
- OS・ミドルウェア管理不要
- 開発に集中できる

---

## 3. クラウドのサービスモデル

### IaaS（Infrastructure as a Service）
**インフラを借りる**

例:
- AWS EC2
- Azure Virtual Machines
- Google Compute Engine

用途:
- 自由度の高いサーバ構築
- 従来システムの移行

---

### PaaS（Platform as a Service）
**実行環境を借りる**

例:
- AWS Elastic Beanstalk
- Azure App Service
- Google App Engine

用途:
- Webアプリ開発
- 環境構築を省略したい場合

---

### SaaS（Software as a Service）
**アプリをそのまま使う**

例:
- Google Workspace
- Salesforce
- Slack

用途:
- 業務ツール
- 開発不要

---

## 4. 主要クラウドベンダー

| ベンダー | 特徴 |
|---|---|
| AWS | 世界最大、サービス数最多 |
| Azure | Microsoft製品と親和性高 |

※ 学習は **AWS から始めるのが定番**。

## 5. クラウドでよく使う主要概念

### ネットワーク
- VPC / サブネット
- セキュリティグループ
- ロードバランサ

### コンピューティング
- 仮想マシン（VM）
- コンテナ（Docker）
- サーバーレス（Lambda）

### ストレージ
- オブジェクトストレージ（S3）
- ブロックストレージ
- ファイルストレージ

### IAM / セキュリティ
- ユーザー・ロール
- 権限管理
- 最小権限の原則

---

## 6. 学習パス（段階別）

### Step 1：基礎理解（概念）
**ゴール**: 用語がわかる

- クラウドとは何か
- IaaS / PaaS / SaaS
- リージョン・AZ
- 従量課金の考え方

---

### Step 2：ハンズオン（触る）
**ゴール**: 実際に動かせる

- EC2 を起動・SSH 接続
- S3 にファイルを置く
- 簡単な Web アプリをデプロイ

---

### Step 3：アプリ構築
**ゴール**: 実務レベルの構成理解

- Web + DB 構成
- ALB + Auto Scaling
- RDS / DynamoDB

---

### Step 4：モダン構成
**ゴール**: 今どきの設計ができる

- Docker / コンテナ
- ECS / Kubernetes
- サーバーレス（Lambda + API Gateway）

---

### Step 5：IaC / 自動化
**ゴール**: 再現性・運用力

- Terraform / CloudFormation
- CI/CD（GitHub Actions）
- 環境のコード管理

---

### Step 6：設計・運用・コスト
**ゴール**: 本番運用を意識できる

- 可用性設計
- 監視（CloudWatch）
- コスト最適化
- セキュリティベストプラクティス

---

## 7. 学習順まとめ（最短ルート）

```
クラウド概念
↓
AWS 基礎
↓
EC2 / S3 / IAM
↓
Webアプリ構築
↓
Docker / Serverless
↓
IaC + CI/CD
```

- スケーラブルなサービス設計
- インフラ構築の自動化
