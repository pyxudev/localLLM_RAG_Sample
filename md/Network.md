# Network

ネットワークの基礎知識とプロトコルについて学習します。

## HTTP と HTTPS

**HTTPS** は HTTP で通信を安全に行うための仕組みです。

- **ポート番号**: 443
- **暗号化**: SSL/TLS
- **特徴**: HTTP はシンプルで汎用性が高い

## IPv6 の特徴

IPv4 にはなく、IPv6 で追加・変更された仕様：

- **アドレス空間**: 128 ビットを割り当てた 16 進数表記
- より広大なアドレス空間により、IoT デバイスなどへの対応が可能

## Web サイト表示の流れ

HTTP を使って Web サイトがブラウザで表示されるまでの流れ：

1. ユーザーがブラウザのアドレスバーに URL を入力する
2. ブラウザが URL を解析し、ドメインから DNS サーバーに Web サーバーの IP アドレスを問い合わせる
3. ブラウザがアクセス先の IP アドレスを取得する
4. ブラウザが Web サーバーにアクセスしてページデータを要求する
5. Web サーバーが要求されたデータを準備する
6. Web サーバーが要求されたデータをブラウザに返信する
7. ブラウザが送られたデータを解析してページとして表示する

## TCP スリーウェイハンドシェイク

TCP 接続確立の手順：

1. **Client → Server**: Request (SYN)
2. **Server → Client**: Response (SYN-ACK)
3. **Client → Server**: Established (ACK)

## マルチプロセッサシステム

複数のプロセッサが主記憶を共用し、単一の OS で制御されるシステムです。

**特徴**:

- システム内のタスクは、基本的にどのプロセッサでも実行できる
- 細かい単位で負荷を分散することで処理能力を向上させる
- スケーラビリティとパフォーマンスの向上が期待できる
